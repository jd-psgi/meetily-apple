Meetily · Template Reference

Meeting Note Templates

Pick a template when you generate a summary, and the on-device model fills each section from your transcript. This is a quick reference to all eleven templates — what each is for, the sections it produces, and an example of the output.

1

Record or import a meeting, then open it and choose Generate Summary.

2

Select a template that matches the meeting. Professional Notes is the default.

3

The summary — and full transcript — are written to your Wiki folder automatically.

General

Professional Notes

Default professional_notes.json

Polished, share-ready notes for almost any meeting. Neutral, professional tone; captures your own “I’ll…” commitments as action items owned by Me. Start here unless a more specific template fits.

Sections

  • Summarytext
  • Key Decisionslist
  • Action Itemstable
  • Discussion Highlightstext

Example output

Summary

The team reviewed Q3 priorities for the Paragon Bank account and agreed to move the Microsoft 365 migration go-live earlier to clear the September FDIC audit. Two MFA-enrollment blockers were identified, with a remediation plan set.

Key Decisions
  • Move M365 go-live to Aug 22 to finish ahead of the audit.
  • Standardize all Paragon endpoints on SentinelOne before migration.
Action Items
OwnerTaskDue
MeSend Paragon the updated migration timelineJul 18
MarcusEnroll remaining 40 users in MFAJul 25
Discussion Highlights

Reviewed audit scope and the risk of legacy shared mailboxes. Priya flagged bandwidth limits at the Collierville branch that could slow the cutover.

  • Incident Summarytext

  • Impacttext
  • Remediation Actions Takenlist
  • Follow-up Action Itemstable

Example output

Incident Summary: Jul 12, 2:14–4:47 AM CDT — Delta Logistics WMS outage. Severity: High. Duration: 2h 33m.

Timeline of Events
TimeEvent
02:14Monitoring alert: WMS host unresponsive
02:40On-call engaged; RAID controller flagged
04:47Service restored from last clean backup

Lessons Learned

Monitoring never alerted on controller-battery health — add that check across all managed hosts.

Formats: text = a written paragraph · list = bullet points · table = a structured table. Templates live as JSON in frontend/src-tauri/templates/ — add or tweak one and it appears in the picker automatically. Every section also obeys the global rule: only what’s actually in the transcript, never invented; empty sections read “None noted in this section.”
